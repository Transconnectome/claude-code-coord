🤔 /coord가 뭔가요?

/coord는 Claude Code의 slash command + skill로, 복잡한 엔지니어링 작업을 6단계 파이프라인으로 자동 조율합니다.

단순한 도구 모음이 아닙니다. 메모리 검색 → 계획 수립 → 병렬 리서치 → 전문가 에이전트 위임 → 리뷰 → 메모리 업데이트까지, 복잡한 작업을 처음부터 끝까지 지능적으로 관리합니다.

✅ 이럴 때 쓰세요

  • 3단계 이상 복잡한 구현
  • 멀티 도메인 아키텍처 설계
  • 원인 불명 복합 버그
  • 대규모 리팩터링 계획

⚡ 이런 효과가 있어요

  • 병렬 에이전트로 시간 절약
  • 이전 세션 메모리 자동 참조
  • 아첨 방지 시스템 내장
  • 전문가 에이전트 자동 배치

📊 6단계 파이프라인 한눈에 보기

🔄 단계별 설명

0⃣ BOOTSTRAP

작업 복잡도 측정 → sequential-thinking + graphiti-memory 자동 로드

1⃣ MEMORY PRE-CHECK

Auto-memory ‖ Serena MCP ‖ Graphiti — 3소스 병렬 스캔

2⃣ PLAN

sequential-thinking MCP → Plan agent → TodoWrite 목록 생성

3⃣ RESEARCH

Explore ‖ deep-research-agent ‖ tavily/context7 병렬 실행

4⃣ EXECUTE

도메인별 전문가 에이전트 병렬 배치 + 중간 lint/test 검증

5⃣ REVIEW + 6⃣ MEMORY

critic agent 검토 → 학습 내용 Auto-memory > Graphiti 저장

Chavis 아첨 방지 시스템 (Phase 5)

Claude Code가 사용자에게 무조건 동의하는 아첨(sycophancy)을 방지하는 5개의 Python hook이 내장되어 있습니다.

🚀 3분 설치법 (Quick Start)

# Step 1 — 저장소 클론

git clone https://github.com/Transconnectome/claude-code-coord.git

# Step 2 — Claude Code 프레임워크 파일 복사

cp -r .claude/ ~/.claude/

# Step 3 — MCP 서버 설치 + 설정

bash scripts/setup.sh

# Step 4 — 사용!

/coord "데이터베이스 스키마 마이그레이션 설계"
